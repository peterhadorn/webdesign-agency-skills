## Design Context

### Users
Developers and technical users looking for answers. They arrive from Google with a specific question and want to find the answer fast. The job: "Find the API method / config option / example I need, right now."

### Brand Personality
Clear, precise, helpful. Documentation should feel like a knowledgeable colleague — not a textbook, not a chatbot.

### Aesthetic Direction
Light theme with optional dark mode. Clean typography with strong heading hierarchy. Generous line height for readability. Sidebar navigation. Code blocks with syntax highlighting. Minimal color — reserve it for links, warnings, and interactive elements.

### Design Principles
1. **Findability over aesthetics** — Search, sidebar nav, and breadcrumbs are critical
2. **Scannable hierarchy** — Users skim headings, not paragraphs
3. **Code is content** — Code blocks are first-class citizens, not afterthoughts
4. **Progressive disclosure** — Show the common case, collapse the edge cases
5. **Cross-device reading** — Must be readable on phone, tablet, and ultrawide
